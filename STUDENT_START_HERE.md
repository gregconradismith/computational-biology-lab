# Start Here for New Students

Welcome to the Computational Biology Lab teaching repository.

This repository is a teaching library rather than a single software package. Most folders contain examples, notes, figures, or workflows that you can study, run, adapt, and extend for your own research projects.

## A Good First Path

If you are new to the lab, begin with this sequence:

1. read the main `README.md` for lab expectations and the repository map
2. decide how you will keep your laboratory notebook
3. choose one technical entry point that matches your current assignment
4. read the folder-level `README.md` before editing or running files
5. keep notes on anything confusing so the next student can benefit from what you learn

## Technical Entry Points

- `Matlab/`: MATLAB examples, worksheets, numerical integration, plotting, fitting, and research-oriented scripts
- `Latex/`: LaTeX examples for manuscripts, posters, and mathematical writing
- `TikZ/`: figure and diagram examples that integrate well with LaTeX
- `Sagemath/`: SageMath and SageTeX examples
- `Notebooks/`: notebook-style materials and exploratory research notes
- `HPC-Sciclone/`: cluster-computing examples for William & Mary's SciClone environment
- `REU-Onboarding/`: arranged notes, assignments, project ideas, tools, and references imported from an Evernote export

## Working Habits

Good research habits matter as much as technical skill.

- Work in small steps.
- Keep a dated laboratory notebook.
- Save commands, parameters, plots, and mistakes that helped you learn something.
- Ask questions early.
- Prefer changes that make the next student's path clearer.

## Before You Edit Shared Examples

Many folders are shared teaching examples. Before reorganizing, renaming, or deleting files, check whether another student may rely on the current path.

When contributing improvements, prefer small focused changes and include enough explanation for someone else to review them.
